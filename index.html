Core Architecture

The Athena Game Server utilizes a bidirectional, persistent WebSocket connection for all backend communication. This architecture completely eliminates the overhead of standard REST APIs and allows the game server to push data instantly.

The Connection Reversal

Unlike traditional setups where a dashboard queries a game server, the Athena plugin acts as the Client. Upon startup, it reaches out to your backend node via ws://127.0.0.1:8000/gameserver. Your backend must act as the Listener.

User & Role Management

The framework implements a priority-based, additive permission system highly inspired by Discord's architecture. Users do not simply have "levels"; they are assigned specific RoleMappings which grant arrays of granular permissions.

Additive Hierarchy

A user's total powers are the union of all roles assigned to them. If Role A grants CanHeal and Role B grants CanSlay, a user holding both can do both.

Priority Enforcement

The system rigorously validates execution priority. A Moderator (Priority 50) is physically blocked from editing, deleting, or granting an Admin role (Priority 100).

The 'Everyone' Role

The system initializes a baseline EveryoneRole. Any permissions added to this role (e.g. CanUnstuck) immediately apply to every player on the server without needing specific assignment.

File Persistence

Role definitions serialize directly to roles.json, while user attachments serialize to user_assignments.json, ensuring full recovery on restart.

Permission Registry

Use the integer IDs below when defining the permissions array in your CREATE_ROLE or UPDATE_ROLE JSON payloads. Alternatively, use the Payload Generator to easily construct standard permission requests.

In-Game Chat Commands

The WebSocket API works in tandem with the native in-game global chat. Players type commands (e.g. /heal or /stuck) directly into the chat, which are seamlessly intercepted.

The Interception Pipeline

When a command starting with a slash (/) is detected in the in-game global chat, the system parses the message, extracts the arguments, and routes it to the designated command handler.

Gatekeeper Verification

Before executing any action, the system checks the required permission for that specific command against the user's assigned roles to ensure they are authorized to perform the action.

Payload Integration

To interface with the endpoints below, connect to the WebSocket listener and transmit JSON strings. The server processes queues sequentially on the main game thread.

NO HTTP ENDPOINTS

There are no URL routes (like /players). You must send a JSON payload with an action property to the main WebSocket URL.

Python 
import asyncio
import websockets
import json

async def connect():
    uri = "ws://127.0.0.1:8000/gameserver"
    async with websockets.connect(uri) as websocket:
        # Mandatory Action Payload
        payload = {
            "action": "SEND_GLOBAL_ANNOUNCEMENT",
            "message": "Hello from Python!",
            "duration": 10.0
        }
        await websocket.send(json.dumps(payload))
        response = await websocket.recv()
        print(f"Response: {response}")

asyncio.run(connect())

Java 
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.WebSocket;
import java.util.concurrent.CompletionStage;
import java.util.concurrent.CountDownLatch;

public class OrionClient {
    public static void main(String[] args) throws Exception {
        CountDownLatch latch = new CountDownLatch(1);
        WebSocket ws = HttpClient.newHttpClient().newWebSocketBuilder()
            .buildAsync(URI.create("ws://127.0.0.1:8000/gameserver"), new WebSocket.Listener() {
                @Override
                public CompletionStage<?> onText(WebSocket webSocket, CharSequence data, boolean last) {
                    System.out.println("Server response: " + data);
                    latch.countDown();
                    return WebSocket.Listener.super.onText(webSocket, data, last);
                }
            }).join();

        String payload = "{\"action\":\"SEND_GLOBAL_ANNOUNCEMENT\",\"message\":\"Hello from Java!\",\"duration\":10.0}";
        ws.sendText(payload, true).join();
        latch.await();
    }
}
Rust 
use tokio_tungstenite::{connect_async, tungstenite::protocol::Message};
use futures_util::{StreamExt, SinkExt};
use serde_json::json;

#[tokio::main]
async fn main() {
    let url = "ws://127.0.0.1:8000/gameserver";
    let (mut ws_stream, _) = connect_async(url).await.expect("Failed to connect");

    let payload = json!({
        "action": "SEND_GLOBAL_ANNOUNCEMENT",
        "message": "Hello from Rust!",
        "duration": 10.0
    });

    ws_stream.send(Message::Text(payload.to_string())).await.unwrap();

    if let Some(msg) = ws_stream.next().await {
        let msg = msg.unwrap();
        println!("Server response: {}", msg.to_text().unwrap());
    }
}

C++ C++ 
#include <ixwebsocket/IXWebSocket.h>
#include <nlohmann/json.hpp>
#include <iostream>

int main() {
    ix::WebSocket webSocket;
    webSocket.setUrl("ws://127.0.0.1:8000/gameserver");

    webSocket.setOnMessageCallback([&webSocket](const ix::WebSocketMessagePtr& msg) {
        if (msg->type == ix::WebSocketMessageType::Open) {
            nlohmann::json payload = {
                {"action", "SEND_GLOBAL_ANNOUNCEMENT"},
                {"message", "Hello from C++!"},
                {"duration", 10.0}
            };
            webSocket.send(payload.dump());
        }
    });

    webSocket.start();
    while (true) { /* Keep thread alive */ }
    return 0;
}
C# C# 
using System;
using System.Net.WebSockets;
using System.Text;
using System.Threading;
using System.Threading.Tasks;

class Program {
    static async Task Main() {
        using var ws = new ClientWebSocket();
        await ws.ConnectAsync(new Uri("ws://127.0.0.1:8000/gameserver"), CancellationToken.None);

        string payload = "{\"action\":\"SEND_GLOBAL_ANNOUNCEMENT\",\"message\":\"C#!\",\"duration\":10.0}";
        var bytes = Encoding.UTF8.GetBytes(payload);
        
        await ws.SendAsync(new ArraySegment(bytes), WebSocketMessageType.Text, true, CancellationToken.None);
    }
}
