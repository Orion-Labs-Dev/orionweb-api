Core Architecture

The Athena Game Server utilizes a bidirectional, persistent WebSocket connection for all backend communication. This architecture completely eliminates the overhead of standard REST APIs and allows the game server to push data instantly.

The Connection Reversal

Unlike traditional setups where a dashboard queries a game server, the Athena plugin acts as the Client. Upon startup, it reaches out to your backend node via ws://127.0.0.1:8000/gameserver. Your backend must act as the Listener. However, if you configure bEnableWebSocketServer=True in the configuration file, the plugin will act as a standard WebSocket server and listen for incoming connections.

Configuration (AthenaMod.ini)

The server automatically generates and maintains an AthenaMod.ini file within the configuration directory. This system is designed to be self-healing: if a server owner deletes a specific key, or an update introduces new features, the mod will automatically inject the missing default keys without overwriting the rest of your custom settings.

User & Role Management

The framework implements a priority-based, additive permission system highly inspired by Discord's architecture. Users do not simply have "levels"; they are assigned specific RoleMappings which grant arrays of granular permissions.

Additive Hierarchy

A user's total powers are the union of all roles assigned to them. If Role A grants CanHeal and Role B grants CanSlay, a user holding both can do both.

Priority Enforcement

The system rigorously validates execution priority. A Moderator (Priority 50) is physically blocked from editing, deleting, or granting an Admin role (Priority 100).

The 'Everyone' Role

The system initializes a baseline EveryoneRole. Any permissions added to this role (e.g. CanUnstuck) immediately apply to every player on the server without needing specific assignment.

File Persistence

Role definitions serialize directly to roles.json, while user attachments serialize to user_assignments.json, ensuring full recovery on restart.

Permission Registry

Use the integer IDs below when defining the permissions array in your CREATE_ROLE or UPDATE_ROLE JSON payloads. Alternatively, use the Payload Generator to easily construct standard permission requests.

In-Game Chat Commands

The WebSocket API works in tandem with the native in-game global chat. Players type commands (e.g. /heal or /stuck) directly into the chat, which are seamlessly intercepted.

The Interception Pipeline

When a command starting with a slash (/) is detected in the in-game global chat, the system parses the message, extracts the arguments, and routes it to the designated command handler.

Gatekeeper Verification

Before executing any action, the system checks the required permission for that specific command against the user's assigned roles to ensure they are authorized to perform the action.

Live Event Streams

The WebSocket connection is fully bidirectional. Not only does the game server accept incoming commands, but it will also continuously push real-time asynchronous events directly through the WebSocket.

Connection Mode Differences

Server Mode (bEnableWebSocketServer=True)

Your external dashboard or bot connects to the game server as a client. It simply needs to establish the connection and listen for incoming messages.

Client Mode (bEnableWebSocketServer=False)

The game server acts as the client and connects outward to your central backend (e.g., OrionWeb). In this mode, your central backend must be set up as a WebSocket Server listener to receive these pushed event payloads.

PARSING REQUIREMENT

These events are pushed automatically and do not include a status or reason field like standard API responses. Always check if type == "log_stream" before parsing.

Event Stream Payload 
{
    "type": "log_stream",
    "stream": "api_command",
    "steamId": "7656119xxxxxxxxxx",
    "action": "CREATE_ROLE",
    "details": "Received from API"
}

Property Breakdown

type

Always perfectly matches "log_stream" for these pushed events. This is the primary key you use to filter these payloads from standard command responses.

stream

The category of the event. The two primary streams are "api_command" (audits when an external API command is received) and "ingame_chat" (live mirrors of player messages sent in global chat).

steamId

The SteamID of the user who triggered the event. If it was triggered by the server itself, this will say "SYSTEM".

action

The specific action taken (e.g., the name of the command executed, or the raw text of the chat message).

details

Additional contextual metadata provided by the server engine.

Payload Integration

To interface with the endpoints below, you must establish a WebSocket connection and transmit JSON strings. Depending on your AthenaMod.ini configuration, your external application will either act as a Client (connecting to the game server) or a Server (listening for the game server to connect to it).

NO HTTP ENDPOINTS

There are no URL routes (like /players). You must send a JSON payload with an action property to the main WebSocket URL.

Connection Mode

Client Mode: The Game Server is acting as the host (bEnableWebSocketServer=True). Your external dashboard, discord bot, or script must establish the connection outward to the game server.

Server Mode: The Game Server is acting as the client (bEnableWebSocketServer=False). You must host a WebSocket Listener on your backend infrastructure to catch the incoming connection from the game server.

import asyncio
import websockets
import json

async def connect():
    uri = "ws://127.0.0.1:8000/gameserver"
    async with websockets.connect(uri) as websocket:
        # Send action payload to the game server
        payload = {
            "action": "SEND_GLOBAL_ANNOUNCEMENT",
            "message": "Hello from Python Client!",
            "duration": 10.0
        }
        await websocket.send(json.dumps(payload))
        response = await websocket.recv()
        print(f"Response: {response}")

asyncio.run(connect())

import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.WebSocket;
import java.util.concurrent.CompletionStage;
import java.util.concurrent.CountDownLatch;

public class OrionClient {
    public static void main(String[] args) throws Exception {
        CountDownLatch latch = new CountDownLatch(1);
        WebSocket ws = HttpClient.newHttpClient().newWebSocketBuilder()
            .buildAsync(URI.create("ws://127.0.0.1:8000/gameserver"), new WebSocket.Listener() {
                @Override
                public CompletionStage<?> onText(WebSocket webSocket, CharSequence data, boolean last) {
                    System.out.println("Server response: " + data);
                    latch.countDown();
                    return WebSocket.Listener.super.onText(webSocket, data, last);
                }
            }).join();

        String payload = "{\"action\":\"SEND_GLOBAL_ANNOUNCEMENT\",\"message\":\"Hello from Java Client!\",\"duration\":10.0}";
        ws.sendText(payload, true).join();
        latch.await();
    }
}
import org.java_websocket.server.WebSocketServer;
import org.java_websocket.WebSocket;
import org.java_websocket.handshake.ClientHandshake;
import java.net.InetSocketAddress;

public class OrionServer extends WebSocketServer {
    public OrionServer(InetSocketAddress address) { super(address); }

    @Override
    public void onOpen(WebSocket conn, ClientHandshake handshake) {
        System.out.println("Game Server Connected!");
        String payload = "{\"action\":\"SEND_GLOBAL_ANNOUNCEMENT\",\"message\":\"Hello from Java Server!\",\"duration\":10.0}";
        conn.send(payload);
    }

    @Override
    public void onMessage(WebSocket conn, String message) {
        System.out.println("Received: " + message);
    }

    @Override public void onClose(WebSocket conn, int code, String reason, boolean remote) {}
    @Override public void onError(WebSocket conn, Exception ex) {}
    @Override public void onStart() {}

    public static void main(String[] args) {
        new OrionServer(new InetSocketAddress("127.0.0.1", 8000)).start();
    }
}

use tokio_tungstenite::{connect_async, tungstenite::protocol::Message};
use futures_util::{StreamExt, SinkExt};
use serde_json::json;

#[tokio::main]
async fn main() {
    let url = "ws://127.0.0.1:8000/gameserver";
    let (mut ws_stream, _) = connect_async(url).await.expect("Failed to connect");

    let payload = json!({
        "action": "SEND_GLOBAL_ANNOUNCEMENT",
        "message": "Hello from Rust Client!",
        "duration": 10.0
    });

    ws_stream.send(Message::Text(payload.to_string())).await.unwrap();

    if let Some(msg) = ws_stream.next().await {
        let msg = msg.unwrap();
        println!("Server response: {}", msg.to_text().unwrap());
    }
}
use tokio::net::TcpListener;
use tokio_tungstenite::accept_async;
use futures_util::{StreamExt, SinkExt};
use serde_json::json;

#[tokio::main]
async fn main() {
    let listener = TcpListener::bind("127.0.0.1:8000").await.expect("Failed to bind");
    println!("Listening on: 127.0.0.1:8000");

    while let Ok((stream, _)) = listener.accept().await {
        let mut ws_stream = accept_async(stream).await.expect("Error during handshake");
        println!("Game Server Connected!");

        let payload = json!({
            "action": "SEND_GLOBAL_ANNOUNCEMENT",
            "message": "Hello from Rust Server!",
            "duration": 10.0
        });
        ws_stream.send(tokio_tungstenite::tungstenite::protocol::Message::Text(payload.to_string())).await.unwrap();

        if let Some(msg) = ws_stream.next().await {
            println!("Received: {}", msg.unwrap().to_text().unwrap());
        }
    }
}

#include <ixwebsocket/IXWebSocketServer.h>
#include <nlohmann/json.hpp>
#include <iostream>

int main() {
    ix::WebSocketServer server(8000, "127.0.0.1");

    server.setOnConnectionCallback([](std::shared_ptr<ix::WebSocket> webSocket, std::shared_ptr<ix::ConnectionState> connectionState) {
        webSocket->setOnMessageCallback([webSocket](const ix::WebSocketMessagePtr& msg) {
            if (msg->type == ix::WebSocketMessageType::Open) {
                std::cout << "Game Server Connected!" << std::endl;
                nlohmann::json payload = {
                    {"action", "SEND_GLOBAL_ANNOUNCEMENT"},
                    {"message", "Hello from C++ Server!"},
                    {"duration", 10.0}
                };
                webSocket->send(payload.dump());
            } else if (msg->type == ix::WebSocketMessageType::Message) {
                std::cout << "Received: " << msg->str << std::endl;
            }
        });
    });

    auto res = server.listen();
    if (!res.first) return 1;
    server.start();
    server.wait();
    return 0;
}

using System;
using System.Net;
using System.Net.WebSockets;
using System.Text;
using System.Threading;
using System.Threading.Tasks;

class Program {
    static async Task Main() {
        HttpListener listener = new HttpListener();
        listener.Prefixes.Add("http://127.0.0.1:8000/gameserver/");
        listener.Start();
        Console.WriteLine("Listening...");

        HttpListenerContext context = await listener.GetContextAsync();
        if (context.Request.IsWebSocketRequest) {
            HttpListenerWebSocketContext wsContext = await context.AcceptWebSocketAsync(null);
            WebSocket ws = wsContext.WebSocket;
            Console.WriteLine("Game Server Connected!");

            string payload = "{\"action\":\"SEND_GLOBAL_ANNOUNCEMENT\",\"message\":\"Hello from C# Server!\",\"duration\":10.0}";
            var bytes = Encoding.UTF8.GetBytes(payload);
            await ws.SendAsync(new ArraySegment(bytes), WebSocketMessageType.Text, true, CancellationToken.None);
            
            var buffer = new byte[1024 * 4];
            while (ws.State == WebSocketState.Open) {
                var result = await ws.ReceiveAsync(new ArraySegment(buffer), CancellationToken.None);
                Console.WriteLine("Received: " + Encoding.UTF8.GetString(buffer, 0, result.Count));
            }
        }
    }
}
